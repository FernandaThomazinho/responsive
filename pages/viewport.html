Viewport

A viewport ou saída de vídeo, em uma tradução livre, é a área de uma página web que é visível ao usuário. A saída de vídeo varia de acordo com o dispositivo e será menor em um mobile do que na tela de um computador. Antes dos tablets e celulares, as páginas web eram projetadas somente para as telas dos computadores e era comum que tivessem um design estático e tamanhos fixos. Então, quando começamos a navegar na internet utilizando dispositivos móveis, tamanhos fixos se tornaram grandes demais para as viewports. Para corrigir isso, os navegadores nestes dispositivos diminuíram proporcionalmente todos os elementos da página para caber na saída de vídeo.
Não foi a solução perfeita, mas uma correção rápida.

Atribuindo a Viewport

HTML 5 introduziu uma maneira de permitir que os web designers controlassem a viewport através da tag meta. Você deve incluir a seguinte tag meta viewport em todas as suas páginas:
<meta name= "viewport" content= "width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no" >

Um elemento meta tag viewport dá instruções ao navegador sobre como controlar as dimensões da página mantendo a proporção. A propriedade width=device-width atribui para a largura da página o comportamento de se adaptar a largura do dispositivo. A propriedade initial-scale=1.0 atribui o nível de zoom com o qual a página será carregada no navegador. Aqui um exemplo de página sem a meta tag viewport e a mesma página com a meta tag viewport:

Usuários geralmente usam barras de rolagem em páginas web, seja em desktops ou em dispositivos móveis, na vertical, mas nunca horizontalmente. Então, se o usuário for forçado a usar a barra de rolagem horizontal, ou diminuir o zoom, para visualizar todo o conteúdo da página isso resultará em uma experiência de uso ruim. Algumas outras regras a seguir:

  1. NUNCA atribua valores fixos na largura dos elementos – Por exemplo, se uma imagem for exibida em uma largura maior que a largura da saída de vídeo isso fará com que a tela fique com barra de rolagem horizontal. Lembre-se de ajustar esse conteúdo para caber na largura da saída de vídeo.
  2. NUNCA dependa de um tamanho de saída de vídeo específico para apresentar bem um conteúdo – Como as dimensões da tela e a largura do vídeo variam muito entre os dispositivos, o conteúdo não deve depender de uma largura de tela específica para ser bem apresentado.
  3. UTILIZE CSS Media Queries para aplicar diferentes estilos para telas pequenas e grandes – Atribuir grandes larguras absolutas para os elementos da tela irá fazer com que eles sejam muito largos para a saída de vídeo de um dispositivo menor. Ao invés disto, considere utilizar larguras relativas, assim como width: 100%.. Cuidado também ao usar posicionamento absoluto com valores muito altos, ou seja, fixar um elemento muito distante da borda esquerda da página. Isso pode fazer com que ele fique para fora em telas menores.
